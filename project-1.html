Ghost Chat - Chat app

What it's about

Using NodeJS and Socket.io I was able to create a chat application which lets you create an account and join a chat room. The user is able to pick an avatar which will display when the user sends a message.

What I used

What I learned

When creating this project the first time I tried using vanilla javascript. I ran into issues with sending messages in real-time. Sent messages would only show up for one user and the other had to refresh the page to see received messages. This was obviously not a viable option. I learned that using Sockets would help me with what I was trying to achieve. Sockets.io enabled the chat windows to update in real-time for all users without the need for refreshing.

Become - Online Store

What it's about

I created this online store which contains a back end and a front end. Clothes items are stored on a Mongoose database and are displayed on the main page of the website. Users are able to add these items to their basket where the data is stored in local storage.

What I used

What I learned

I came across several issues when developing this web store. Firstly I had issues storing the basket items in local storage after page refresh. Instead of storing the HTML elements of each basket item I was trying to push these items to an array. This caused the basket to display incorrectly each time. Sometimes newly added items would replace older items instead of adding to the current basket.

I also noticed how difficult it can be to create a webstore using vanilla JavaScript instead of a framework like React. A lot of the code I wrote was reused several times because the page needs to reload as a whole instead of loading only one component like in React..

What I used

What I learned
